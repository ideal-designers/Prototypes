Інструкція старту нової сесії

Покрокове керівництво для швидкого запуску Claude Code з повним контекстом проекту

Автоматично: Claude Code читає CLAUDE.md при кожному старті і бачить директиву прочитати SKILL.md. Але краще продублювати запит у першій репліці для надійності.

Кроки запуску

1
Перша репліка — завантаження бази знань
Відправ цю команду одразу після старту сесії. Claude прочитає SKILL.md і буде знати всі токени, компоненти, шаблони.
// Перша репліка в новій сесії Прочитай SKILL.md
2
Задача з Figma — реалізація прототипу
Передай Figma URL і slug прототипу. Claude одразу готовий до роботи.
// Шаблон для нового прототипу Figma: https://www.figma.com/design/... Прототип: my-prototype-slug Завдання: реалізувати [опис UI]
3
Створення нового прототипу (scaffold)
Для генерації нового прототипу використовуй скрипт. Claude сам запустить і закомітить.
node scripts/new-proto.js \ --slug "my-slug" \ --title "My Prototype Title" \ --description "Short description" \ --figma "https://figma.com/design/..."
4
Дебаг через Chrome DevTools MCP
MCP підключений — можна дивитися консоль, помилки мережі, робити скріни прямо в сесії.
// Перегляд помилок на живому сайті Відкрий https://prototype-dashboard-sigma.vercel.app/my-slug і покажи console errors

Ключові файли проекту

Файл Призначення
CLAUDE.md Автозавантаження при старті сесії. Правила іконок, git.
SKILL.md Повна база знань DS: токени, компоненти, шаблони, патерни.
src/app/shared/ds/ DS компоненти, іконки, токени (tokens.css).
src/app/prototypes/ Всі прототипи. Кожен — окремий Angular компонент.
src/app/proto-registry.ts Реєстр прототипів (slug, title, figma, status).
scripts/new-proto.js Scaffold нового прототипу з правильним шаблоном.
.mcp.json Chrome DevTools MCP — дебаг браузера з сесії.
